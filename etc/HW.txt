: STRIDE : 소프트웨어에 집중하는듯
	: 위장(spoofing)
		: Diagnosis Server 주소 사칭할 수 있으려나 흠
	: 변조(tampering)
	: 부인(repudiation)
		: ID가 위조됐다는 등의 주장? 양심없는 사람들의 경우 박박 우길순 있겠다.
	: 정보 노출(information disclosure)
		: 포커스 맞춰야 할 부분인듯, 프라이버시?
		: 이론적으로는 안될거같긴한데..
	: 서비스 거부(denial of service)
		: 누가 난수들어간 블루투스 패킷 뿌리고다니면 곤란해질수도..
	: 권한 상승(elevation of priviledge)
		: 어플에서 취약점이 발견되거나 하면?
: LINDDUN : 프라이버시에 집중, DFD로 표현된 외부 객체, 프로세스, 데이터 저장소, 데이터 흐름에 대한 위협 식별 후 각 위협을 위협트리로 작성해 개인정보위협을 체계화 + 시나리오
	: 연결(linkability)
		: Tracing Key, Daily Tracing Key는 어떤 일방향함수를 사용하는지
			: RPI(Rolling Proximity ID)는 어떻게 계산되는지 등
	: 식별(identifiability)
		: 얘는 연결이랑 차이가 뭔지;
	: 부인 방지(non-repudiation) : STRIDE에도 있음
	: 검출(detectability)
		: 위치정보정도? 이론적으로는 신원 식별이 안돼서 ㄱㅊ할듯
	: 정보 노출(information disclosure) : STRIDE에도 있음
