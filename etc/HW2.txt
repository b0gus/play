* Security Analysis of the COVID-19 Contact Tracing Specifications by Apple Inc. and Google Inc.
- propose mitigation stratigies that do not use challenge-response protocols / a public key infrastructure(?)
- Overview of the Specifications 
: 참고 (3개의 스펙문서 요약)
	: 참고문헌에 다 있음
- System Security Analysis
: Power and Storage Drain Attacks
	: DoS 공격에 포함, 밀집공간에서 위력적일듯
	: advertisement를 계속 보내서 배터리를 소모시킴
	: 또한 타겟이 되는 디바이스를 느리게해 걔가 advertisement를 못할수도..
	: 스펙 문서에 DoS 공격에 대한 보호 방법에 있다고는 언급되어있으나, 방법은 설명하지 않음 (중복되는 메시지는 필터한다는 정도)
	: 문제 : 배터리 고갈을 경험한 유저는 시스템 사용을 취소할 가능성이 높으며, 많은 유저가 사용하지 않으면 시스템의 효용성이 현저히 낮아질 것
	: mitigation
		: 공격의 시간, 신호 강도 등의 정보를 클라우드에 전송하고 이러한 메시지가 많아지면 공격자의 위치를 식별해 적절한 조치를 취할 수 있을 것
		: 다량의 advertisement를 수행할 경우, 디바이스를 깨우지 않고 하드웨어적으로 proof-of-work를 수행하도록 함으로써 DoS 공격이 불가능하도록(?) - 소량의 계산
: Relay and Replay Attacks
	: 타 유저에게 수신한 advertisement 메시지를 전달(사칭?)
	: 문제 : 확진자를 사칭한 공격이 진행될 경우, 많은 사람들이 착각해 검진을 받게 됨으로써 자원이 소모될 것이며 시스템에 대한 신뢰도가 하락할 수 있음
	:  mitigation
		: challenge-response 프로토콜 사용? - 아키텍처의 변화가 필요해 비효율적
		: self-locating(?) - applendix에 있다는듯
			: location cell 내에서만 공격 진행할 수 있도록 범위 제한
		: 수식들어간 방법 제시..
: Trolling Attacks
	: false positive 유도 (감염안된사람이 감염됐다고 믿게 하려하기 위한 공격)
	: 감염된 공격자가 자신의 폰을 드론이나 차에 매달고 돌아다니게 함으로써 다른 사람들에게 경고메시지가 가도록 한다는 등의 공격 형태
	: 문제 : 불필요한 검진을 받는 사람들 증가, 시스템과 질병에 대한 신뢰도 하락 등
	: mitigation
		: 공공의료기관에 확진자 RPI(?) 제공
		: abnomal trace를 통해 탐지하는 방법 등
- Information Security Analysis
: Linking Attacks
	: 수집 가능한 정보들 사이 링크를 생성해 프라이버시 위협 (시간, 신호 강도 등)
	: 문제 : 개인을 특정한다던지 하는 수준의 위협이 발생한다면, 사람들이 시스템을 사용하지 않을 것
	: mitigation : 신호 강도, advertisement 빈도 등을 변화시키며 공격자가 디바이스의 위치를 유추하기 어렵도록 함
: Tracking and Deanonymization Attacks
	: signal-locating attack
		: 공격자는 블루투스 신호 강도와 공격자의 기기의 위치를 통해 특정 디바이스 위치 유추
		: mitigation : 신호 강도를 변화시키도록 조정
	: singling-out attack
		: 공격자가 디바이스를 어떤 사람 근처에 두고 RPI 수신, 이후 그 사람이 확진판정을 받으면 알람이 올테고 그러면 확진자가 누구인지 알아낼 수 있음
		: 현실성이 조금 떨어져보이긴함;
		: mitigation도 언급되어있는데 일단 보류..
